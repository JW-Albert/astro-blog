# CLAUDE.md

This file provides guidance for Claude Code when working in this repository.

## Project Overview

An Astro blog deployed as a static site on **Cloudflare Workers** via `wrangler`. Supports bilingual content (English / Traditional Chinese) through a client-side language toggle.

## Key Commands

| Command | Purpose |
|---------|---------|
| `npm run dev` | Local dev server at `localhost:4321` |
| `npm run build` | Build to `./dist/` |
| `npm run preview` | Build then run locally via Wrangler |
| `npm run check` | Full check: build + tsc + wrangler dry-run |
| `npm run deploy` | Deploy to Cloudflare Workers via Wrangler |

Always run `npm run build` before committing to confirm the build succeeds. The project uses `node >=22`.

## Bilingual System

Language state is stored in `localStorage` under the key `lang` (`"en"` or `"zh-TW"`). The `<html lang>` attribute is toggled by `LangToggle.astro`, and an inline script in `BaseHead.astro` applies the stored language before first paint to avoid flash.

**CSS rules** in `global.css`:
```css
html[lang="zh-TW"] [data-en] { display: none; }
html[lang="en"]    [data-zh] { display: none; }
```

### Adding bilingual content

**In `.astro` pages** — wrap text in `<span>` or `<div>`:
```html
<h1>
  <span data-en>Hello</span>
  <span data-zh>你好</span>
</h1>
```

**In `.mdx` blog posts** — use block-level divs with blank lines inside:
```mdx
<div data-en>

English paragraph content here.

</div>

<div data-zh>

中文段落內容。

</div>
```

### Blog post frontmatter

```yaml
---
title: "English Title"
titleZh: "中文標題"          # optional — shown in post header and blog list
description: "English desc"
descriptionZh: "中文描述"    # optional
pubDate: "Apr 10 2026"
updatedDate: "Apr 11 2026"   # optional
heroImage: "/blog-placeholder-1.jpg"  # optional, placed in public/
---
```

## Important Conventions

- **All blog posts must be `.mdx`** — `.md` files are not used. The content collection glob picks up `**/*.{md,mdx}` but only `.mdx` supports the bilingual `<div>` blocks.
- **MDX requires self-closing void tags**: use `<br/>` not `<br>`, `<img/>` not `<img>`.
- The `title` field is used for SEO (`<title>`, OpenGraph). Keep it in English; use `titleZh` for the Chinese display title.
- Do not edit `dist/` — it is generated output.
- Do not modify `worker-configuration.d.ts` — it is auto-generated by `npm run cf-typegen`.
